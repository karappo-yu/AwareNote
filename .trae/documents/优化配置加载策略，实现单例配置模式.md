# 优化配置加载策略，实现单例配置模式

## 问题分析
当前代码中，`pdf_utils.py` 文件在多个函数中重复读取 `setting.json` 配置文件，每次调用都会产生额外的IO操作，影响性能。而 `config/config.py` 已经实现了配置的单例模式，`image_utils.py` 和 `scanner/scanner.py` 都已经正确使用了它，但 `pdf_utils.py` 没有使用。

## 解决方案
修改 `pdf_utils.py` 文件，使其使用 `config/config.py` 中的配置缓存，而不是直接读取配置文件，从而减少IO操作，提高性能。

## 实现步骤

### 1. 修改 pdf_utils.py 文件
- 移除所有直接读取配置文件的代码
- 从 `config.config` 导入所需的配置项
- 使用 `config.get()` 函数获取其他配置项

### 2. 具体修改点

#### 2.1 导入配置模块
在文件顶部添加：
```python
from config.config import get as get_config, compressed_width
```

#### 2.2 修改 pdf_to_images 函数
- 移除 `config_path` 和配置加载代码
- 使用 `compressed_width` 变量替代直接读取的配置

#### 2.3 修改 get_pdf_page_image 函数
- 移除 `config_path` 和配置加载代码
- 使用 `compressed_width` 变量替代直接读取的配置

#### 2.4 修改 pdf_to_images_async 函数
- 移除 `config_path` 和配置加载代码
- 使用 `compressed_width` 变量替代直接读取的配置

#### 2.5 修改 get_pdf_page_image_async 函数
- 移除 `config_path` 和配置加载代码
- 使用 `compressed_width` 变量替代直接读取的配置

### 3. 验证
- 确保所有配置读取都通过 `config/config.py` 模块进行
- 验证代码功能正常
- 测试性能是否有所提升

## 预期效果
- 减少配置文件的IO操作，提高性能
- 实现配置的单例模式，确保配置一致性
- 代码更加简洁，易于维护
- 保持向后兼容性，不影响现有功能

## 风险评估
- 低风险：修改仅涉及配置加载方式，不影响核心功能
- 保持向后兼容：所有配置项的获取方式保持不变
- 配置一致性：使用统一的配置源，避免配置不一致的问题
